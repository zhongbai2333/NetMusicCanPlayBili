# MC 26.1 (1.21.2) 渲染 API 调研

## 重大变更

### 1. OpenGL → Vulkan 过渡 (26.2)
> "26.1 is expected to be the last Minecraft release to solely support OpenGL"
> — FabricMC blog, 2026-03-14

**结论: 绝对不能用 raw GL 调用。必须走 Blaze3D API。**

### 2. 已废弃的 API (26.1 移除)
- `glBegin/glEnd` (OpenGL 3.3 Core Profile 禁用)
- `RenderSystem.setShader()` (shader 系统重做)
- `RenderType` (改名/拆分)
- `BufferBuilder.addVertex(x, y, z)` → 需要 Matrix4f
- `Tesselator` API 变化

### 3. 新 API: Subclass-based RenderLevelStageEvent

旧 (1.21.0):
```java
event.getStage() == RenderLevelStageEvent.Stage.AFTER_TRANSLUCENT
```

新 (26.1):
```java
// 每个阶段是独立子类
RenderLevelStageEvent.AfterSky
RenderLevelStageEvent.AfterOpaqueBlocks
RenderLevelStageEvent.AfterEntities
RenderLevelStageEvent.AfterTranslucentBlocks  ← 用这个
RenderLevelStageEvent.AfterTripwireBlocks
RenderLevelStageEvent.AfterParticles
RenderLevelStageEvent.AfterWeather
RenderLevelStageEvent.AfterLevel
```

事件只提供:
- `getPoseStack()`
- `getLevelRenderer()`
- `getLevelRenderState()`
- `getModelViewMatrix()`
- `getRenderableSections()`

**不提供** MultiBufferSource, CameraRenderState 等。

### 4. BER 渲染新 API (SubmitNodeCollector)

26.1 的 BER 使用特征提交模型：

```java
// 核心 API
collector.submitCustomGeometry(
    poseStack,                          // PoseStack
    RenderTypes.text(atlasLocation),    // RenderType (plural!)
    (pose, consumer) -> {               // VertexSubmission lambda
        consumer.addVertex(pose, x, y, z)  // Matrix4f + float coordinates
            .setColor(0xFFFFFFFF)          // ARGB
            .setUv(0, 1)                   // UV
            .setLight(packedLight);        // Lightmap
    }
);
```

## 视频渲染方案

### 纹理管理
- 视频帧上传: `glTexImage2D` + `glTexSubImage2D` ✅ (Blaze3D 底层仍用 GL)
- **必须注册到 TextureManager** — 不能裸用 GL texture ID
- 用 `DynamicTexture` 或 `NativeImageBackedTexture`

### 世界内渲染
两条路:
1. **BER 方案** — 创建 BlockEntity + BER，用 `submitCustomGeometry`
2. **RenderLevelStageEvent 方案** — AfterTranslucentBlocks + `mc.renderBuffers().bufferSource()`

推荐 BER 方案（已在自己的 ModernTurntableRenderer 验证可行）。

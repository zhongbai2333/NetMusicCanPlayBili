# Pad 设计文档

Pad 是 MP4 之后的第二个手持媒体设备。它不是"播放列表 MP4"，而是一个**地图式定点媒体触发器**：创作者在 Pad 的地图界面上创建触发点位，把物品内的 NetMusic 唱片按稳定编号分配给点位；使用者点击点位或进入点位范围后，在 Pad 的 GUI 屏幕内触发播放。所有交互均发生在 Pad 物品的 3D 渲染表面上，不弹出全屏地图窗口。

## 核心定位

- MP4：列表播放器，围绕 `selectedQueueIndex` 播放。
- Pad：地图点位播放器，围绕 `pointId -> mediaId -> disc` 播放。
- Pad 的视频播放目标是 **Pad GUI 内播放**，视觉形态接近 MP4 的横屏播放模式（视频渲染在 Pad 3D 表面的播放区域上，不切换额外界面）。
- Pad 仍然要支持耳机与全息眼镜绑定：
  - 耳机接收 Pad 当前触发媒体的音频。
  - 全息眼镜接收 Pad 当前触发媒体的视频 HUD/全息显示。

## 默认界面

Pad 的界面渲染在物品的 3D 模型表面上（`submitCustomGeometry`），不是独立的全屏 GUI。所有交互——点击点位、拖放歌曲——都通过覆盖在渲染面之上的透明输入层（`PadFocusScreen`）捕获后投影到纹理坐标完成，不弹出额外地图窗口。

地图**不支持拖拽平移**，始终以玩家当前位置为中心，省去额外的视口计算和缓存刷新开销，保证交互的流畅性。地图区域显示玩家附近地形的风格化分类：

- `GRASS`：草方块、泥土、苔藓、草地植被。
- `BUILDING`：屋顶/墙体等有室内空间的垂直结构（基于几何证据判定：顶面下方有连续空腔，或顶面显著高于邻近地表）。
- `WATER`：水、冰。
- `TREE`：原木 + 树叶。
- `FARMLAND`：耕地、农作物。
- `ROCK`：岩石、砂砾等自然裸岩。
- `SNOW`：雪。
- `INDOOR_FLOOR`：室内地面（仅室内视图）。

> 不计划实现道路（`ROAD`）分类及道路导航功能。贴地的人工铺装（石砖、混凝土路等）按普通地面处理。未来若引入道路识别，预计需要 CNN 级别的算法支持。

点位在地图上以简洁的 pin 符号标记，符号下方显示该点位绑定的歌曲名称。编辑状态下用两种颜色区分可见性：红色 pin = 发布后对用户可见，灰色半透明 pin = 隐藏点位（发布后不显示，仅触发半径生效）。锁定（发布）状态下仅渲染可见点位。

地图右侧（或右上区域）显示**媒体库面板**，列出 Pad 中所有已装入的唱片（按 `mediaId` 排列），显示歌曲名称和编号。创作者可以直接从面板中拖拽歌曲到地图上完成点位创建。

后处理：连通域合并、小区域噪声过滤、边界平滑/描边，让地图更像现代地图 APP。

## 创作者编辑流程

Pad 类似原版书与笔，分为草稿和锁定状态。

### 拖放式点位创建

核心交互模式：**从媒体库面板拖拽歌曲到地图上**。无需单独的"添加点位"按钮或编辑模式切换。

1. 打开 Pad，地图以玩家位置居中显示，右侧显示媒体库面板。
2. 在媒体库中找到要绑定的歌曲，按住拖拽到地图上的目标位置。
3. 松开后在该位置创建触发点，默认绑定该歌曲的 `mediaId`。
4. 点位创建后可点击展开编辑卡片：修改名称、触发半径、触发方式（手动 / 进入半径）、循环开关、音量，以及**是否在发布后的用户界面中显示该点位**。
5. 点位支持拖拽微调位置（地图上的 pin 可直接移动）。
6. 右键点击已有点位可删除。
7. 保存后将修改写入物品 NBT。

### 草稿状态

草稿状态允许自由编辑：

- 从媒体库拖入新歌曲到地图创建点位。
- 修改已有 pin 的位置、名称、绑定、触发参数。
- 删除点位。
- 向媒体库添加/移除唱片（通过物品栏右键交互，与当前实现一致）。

### 锁定状态

锁定后类似已签名书：

- 不能继续编辑点位和媒体库。
- 只能浏览、点击点位触发播放、绑定耳机/全息眼镜。
- 地图仍以玩家为中心跟随，但不再响应拖放操作。

## 媒体库模型

Pad 不使用 MP4 的播放列表模式，而是媒体编号库。

```text
mediaId -> NetMusic 唱片
```

规则：

- 最大媒体数初版为 64。
- 添加唱片时分配最小空闲 `mediaId`。
- 删除唱片时不重排编号。
- 点位绑定缺失媒体时显示“缺失媒体”，不自动改绑。

这样可以保证点位绑定长期稳定。

## 点位模型

点位以世界坐标为权威，地图坐标只是 UI 投影结果。

```text
pointId: UUID
name: String
x/y/z: double
radius: int
mediaId: int
triggerMode: MANUAL | ENTER_RADIUS
loop: boolean
volumePerMille: int
visible: boolean
```

### 点位可见性

`visible` 字段控制该点位在锁定（发布）后的用户界面中是否显示 pin 图标和歌曲名：

- `visible = true`（默认）：明确点位 —— pin 和名称正常显示，用户可直接看到并点击触发。适合主路引导、公开曲目。
- `visible = false`：隐藏点位 —— 地图上不渲染 pin 和名称，但触发半径仍然生效。用户需要靠探索走到范围内才能触发。适合彩蛋、解密、隐藏奖励。

编辑状态下所有点位始终可见（含隐藏点位），用不同透明度或虚线边框区分，方便创作者管理。

### 触发方式

- `MANUAL`：点击点位播放。
- `ENTER_RADIUS`：进入半径触发（后续阶段实现）。

## 播放规则

触发播放时：

```text
pointId -> mediaId -> disc -> resolve -> playback session
```

播放行为取决于媒体类型：

- **视频媒体**：默认以全屏模式渲染在 Pad 3D 表面的播放区域上，覆盖地图界面。玩家可手动退出全屏回到地图视图，此时视频缩为小窗或暂停（取决于后续设计）。全屏状态下地图上的点位仍可点击触发。
- **纯音频媒体**：不占据地图界面，仅在底部或侧边弹出一个精简的进度条 + 播放控制（播放/暂停、上一首/下一首），地图仍可正常浏览和交互。

与 MP4 一致的约束：

- 视频解码仅在 Pad 处于玩家 **hotbar 或副手**时持续进行；切换至其他物品或关闭背包后暂停解码以节省资源。
- 音频播放不受 hotbar 限制，切换物品后继续播放（与 MP4 行为一致）。
- 耳机和全息眼镜绑定逻辑沿用 MP4 的设备链路（`MediaBindingData` / `AudioLinkIndex`），Pad 的 `deviceId` 作为媒体源标识。

声音和空间信息仍可使用点位坐标，便于耳机范围与未来世界点位声源扩展。

## 同步模型

Pad 沿用 MP4 当前配置表同步原则：

- DeviceID 独立分配/下发。
- 客户端维护本地 PadDocument。
- 服务端维护权威 PadDocument。
- 小同步 10t，客户端和服务端都可请求。
- 小同步合并 10t 周期内所有 dirty 操作，只上传最后文档。
- 大同步 300t，服务端权威下发。
- 同步包带 `updatedAtMillis + sequence`，服务端拒绝旧同步。

## 数据结构

Pad 物品 CustomData 规划：

```text
pad_device_id
pad_title
pad_author
pad_locked
pad_media_entries
pad_trigger_points
pad_map_settings
```

### 设计偏移记录

以下原始计划已被修订或废弃，不再作为开发目标：

| 原始计划 | 当前决定 | 原因 |
|---|---|---|
| `ROAD` 道路分类 | 删除 | 启发式算法准确率不足，未来需要 CNN |
| 地图可拖拽平移 | 固定以玩家为中心 | 减少视口计算和缓存刷新开销 |
| 点击地图区域 → 打开独立 `PadMapScreen` | 废弃独立地图 GUI | 交互统一在 Pad 3D 表面完成 |
| "添加点位"按钮 → 创建空 pin → 绑定唱片 | 从媒体库面板拖拽歌曲到地图 | 更直观，pin 创建即绑定 |
| 点位无可见性概念 | 新增 `visible` 字段 | 支持彩蛋/解密玩法（隐藏 pin） |
| 视频"横屏播放布局" | 视频全屏覆盖地图 + 可手动退出；纯音频仅进度条 | 区分媒体类型，音频不遮挡地图 |

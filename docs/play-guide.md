# Net Music Can Play Bili 游玩指南

> 本指南适用于 `0.4.1-beta`。模组仍处于测试阶段，界面与玩法可能在后续版本中调整。

Net Music Can Play Bili 是 NetMusic 的 NeoForge 附属模组。它让你可以把 Bilibili 视频刻录为 NetMusic 唱片，并通过现代化唱片机、投影仪、音响、MP4、Pad、耳机和全息眼镜组成一套完整的游戏内媒体系统。

## 安装要求

| 项目 | 要求 |
| --- | --- |
| Minecraft | 26.1.2 |
| NeoForge | 26.x，开发版本为 26.1.2.76 |
| Java | 25 |
| NetMusic | 1.5.1 或更高版本，必装 |
| Curios | 12 或更高版本，可选 |

- 单人游戏：把本模组和 NetMusic 放进客户端的 `mods` 文件夹。
- 多人游戏：客户端和服务端都需要安装本模组及 NetMusic，且版本应保持一致。
- 安装 Curios 后，可以把耳机和全息眼镜放进饰品头部槽；未安装时，它们使用原版头盔栏。

## 五分钟快速开始

### 1. 登录 Bilibili

1. 按 NetMusic 的玩法制作电脑和 CD 刻录机。
2. 打开 NetMusic 电脑。
3. 点击 **B站登录**，使用哔哩哔哩手机客户端扫码登录。
4. 登录后可点击 **杜比：开/关**，决定刻录时是否优先尝试 Dolby 音频。

登录可以提高部分视频和高画质资源的解析成功率。登录信息保存在本地配置中，请勿把配置文件发给其他人。

### 2. 刻录一张 B站唱片

打开 NetMusic CD 刻录机，在链接输入框中填写以下任一种内容：

- `BV...` 视频号；
- `av...` 视频号；
- 包含 BV 号或 av 号的 Bilibili 视频链接。

多 P 视频可在 **B站分P** 输入框中填写分 P 序号，留空时默认使用第 1 P。序号必须是大于等于 1 的整数。

刻录完成后会得到一张 NetMusic 音乐 CD。它是本模组绝大多数播放设备的媒体来源。

> NetMusic 的广播喇叭还支持输入 `live:直播间ID` 播放 Bilibili 直播音频。

### 3. 使用现代化唱片机

1. 放置现代化唱片机。
2. 手持刻录好的 NetMusic 唱片，右键唱片机：插入唱片并开始播放。
3. 唱片机内已有唱片时，普通右键可弹出唱片。
4. **Shift + 右键**唱片机可打开控制界面。

控制界面支持播放/暂停、重播、单曲循环、拖动进度和调节本地音量。

### 唱片机自动化与红石

现代化唱片机支持漏斗及兼容 NeoForge 物品传输接口的管道：

- 从任意方向向空唱片机输入一张有效的 NetMusic 唱片，唱片在传输完成后会自动开始播放；
- 唱片机一次只能容纳一张唱片，非 NetMusic 唱片不会被接收；
- 漏斗或管道可以从唱片机中取出唱片，取出后当前播放会停止；
- 自动输入唱片后，是否立即播放还会遵循当前红石模式。

在 **Shift + 右键**打开的控制界面中，点击“红石”按钮可按以下顺序切换三挡模式：

| 红石模式 | 行为 |
| --- | --- |
| 高信号 | 有红石信号时开始或继续播放；失去信号时暂停 |
| 低信号 | 有红石信号时暂停；失去信号时开始或继续播放 |
| 忽略红石 | 不受红石电平变化影响，插入唱片后正常自动播放，也可手动播放或暂停 |

模式切换后会立即按唱片机当前收到的红石电平执行一次，之后每次电平变化都会更新播放状态。高信号和低信号模式都从暂停位置继续，不会因为切换电平而从头播放；当当前电平要求停止时，手动播放或重播不会绕过红石模式，当当前电平要求播放时，手动暂停也会由模式接管。曲目自然播放结束不会因信号持续不变而自动重播。旧世界中的唱片机默认使用“忽略红石”，以保持原有玩法。

在唱片机旁连接红石比较器，可以把播放进度转换成红石信号：

| 唱片机状态 | 比较器输出 |
| --- | --- |
| 空唱片机或尚无有效曲目信息 | 0 |
| 曲目刚开始 | 1 |
| 播放中 | 随播放进度逐步从 1 增加到 15 |
| 临近曲目结尾 | 15 |
| 播放自然停止后 | 0 |
| 暂停 | 保持暂停时的进度强度 |

利用比较器输出可以制作进度灯带、曲终触发器或自动换片装置。比较器显示的是整首曲目的相对播放进度，不是唱片机物品槽的填充程度。

## 搭建影音系统

现代化唱片机可以连接歌词投影仪、视频投影仪和音响。最方便的做法是先绑定，再放置：

1. 把尚未放置的歌词投影仪、视频投影仪或音响拿在主手。
2. 普通右键现代化唱片机。
3. 出现“已存储连接目标”提示后，把设备放到想要的位置。
4. 设备放下时会自动连接刚才的唱片机。

绑定关系按维度保存，请把唱片机和设备放在同一维度。需要重新布置时，重新拿设备物品绑定即可。

### 歌词投影仪

右键已放置的歌词投影仪可打开设置界面，调整：

- 投影朝向、俯仰、高度和水平偏移；
- 画面缩放；
- 静态、轮换·主、轮换·副显示模式；
- AI 字幕开关。

如果原视频没有可用字幕，可以尝试 AI 字幕；生成和显示效果取决于媒体内容与解析结果。

### 视频投影仪

右键视频投影仪可调整朝向、俯仰、高度、X/Z 偏移、缩放和画质。

可尝试的画质包括 360P、480P、720P、1080P、1080P+、1080P60、4K 和 8K。实际能否取得对应画质，取决于视频源、登录状态、账号权限和网络状况。手持设备或较小画面一般使用 720P 或 1080P 即可，更高分辨率只会带来有限的观感提升，却会显著增加解码和 GPU 压力。

### 音响

右键音响可设置：

- 7.1.4 声道位置；
- 静音；
- 是否融合未分配声道；
- 0～2 倍音量。

一个配方会产出两个音响。摆放多个音响并分配不同声道，可以获得更明显的空间音频效果。

## MP4

MP4 是随身播放列表，最多可装入 18 张 NetMusic 唱片。

### 装入和取出唱片

1. 打开物品栏。
2. 用鼠标拿起一张 NetMusic 唱片。
3. 左键点击 MP4，把唱片加入播放队列。
4. 鼠标为空时右键点击 MP4，可取出当前选中的唱片；没有有效选择时取出最后一张。

### 播放操作

手持 MP4 右键进入聚焦界面。界面支持：

- 上一首、下一首和播放/暂停；
- 查看播放列表；
- 视频开关与横屏显示；
- 画质选择；
- 字幕模式与 AI 字幕；
- 音量和循环播放。

视频解码只会在 MP4 位于快捷栏或副手等活跃位置时持续进行，以减少不必要的资源消耗；切换物品后，音频仍可继续播放。

## Pad 地图玩法

Pad 不是普通播放列表，而是一张可以布置“音乐点位”的地图。它适合制作景区导览、剧情配乐、解谜和彩蛋。

### 制作草稿

1. 在物品栏中，把 NetMusic 唱片左键放到草稿 Pad 上。
2. 手持 Pad 右键，打开显示在 Pad 表面的地图界面。
3. 从右侧媒体库按住一首歌曲，拖到地图上的目标位置。
4. 松开后会创建一个 Pin，并绑定这首歌曲。
5. 选择 Pin 后，可调整显示状态、触发半径、触发方式、循环和音量，也可删除或拖动 Pin 微调位置。

Pad 地图始终以玩家为中心，不支持拖拽平移。

### 点位类型

- **手动触发**：使用者点击地图上的 Pin 后播放。
- **进入范围触发**：使用者进入设定半径后自动播放。
- **可见点位**：发布后显示 Pin 和歌曲名称。
- **隐藏点位**：发布后不显示图标，但进入触发范围仍然生效，适合制作彩蛋。

### 发布副本

编辑完成后，点击 Pad 底部的 **发布副本**：

- 背包中会获得一个锁定副本；
- 原草稿仍可继续编辑；
- 锁定副本只能浏览和触发，不能修改媒体库或点位；
- 草稿与发布副本共享同一个设备文档，草稿后续修改会同步到对应副本。

Pad 最多保存 64 个媒体和 128 个点位，点位触发半径可设为 1～128 格。

## 耳机、全息眼镜与绑定

### 穿戴设备

- 主手拿隐形耳机、猫耳无线耳机或全息眼镜右键，可直接装备到空的头盔栏。
- 如果头盔栏已有装备，请先取下头盔，或安装 Curios 并使用饰品头部槽。
- 带有“耳机”或“全息眼镜”能力的头部装备也可以用于绑定。

耳机把绑定设备的声音私有地传给佩戴者。全息眼镜会在本地隐藏普通视频投影画面，并将绑定媒体显示在 HUD 全息层中；最多可绑定 4 个媒体源。

### 使用媒体管理工具绑定

媒体管理工具负责把唱片机、MP4 或 Pad 与耳机、全息眼镜连接起来。

#### 绑定唱片机

1. 主手拿媒体管理工具，右键目标唱片机。
2. 在界面右上槽放入耳机、全息眼镜或具有对应能力的装备。
3. 点击向下箭头确认绑定。

#### 绑定 MP4 或 Pad

1. 主手拿媒体管理工具。
2. 在工具界面的左侧槽放入 MP4；绑定 Pad 时，也可副手拿 Pad 后右键空气进入对应绑定流程。
3. 在右上槽放入要绑定的耳机或全息眼镜。
4. 点击向下箭头确认绑定。

点击 **解绑** 可清除目标设备或装备上的绑定。

需要注意：

- 唱片机已经连接音响时，不能再把耳机连接到该唱片机；
- 唱片机已经连接视频投影仪时，不能再把全息眼镜连接到该唱片机；
- 全息眼镜不直接绑定视频投影仪，应绑定唱片机、MP4 或 Pad；
- 耳机接收范围约为 64 格，并且玩家与设备必须在同一维度。

### 全息眼镜快捷键

佩戴全息眼镜后，按 `H` 打开配置界面。按键冲突时可在 Minecraft 的控制设置中重新绑定。

## 媒体管理与举报

在多人服务器中，媒体管理工具也可以举报正在播放的音源：

- **Shift + 右键空气**：列出附近可听到的活跃音源；
- **Shift + 右键唱片机**：直接举报该唱片机；
- 副手拿 MP4 时 **Shift + 右键**：举报该 MP4 音源。

举报会发送给在线的 OP2+ 管理员；如果没有管理员在线，记录仍会被保存。重复举报同一音源会自动合并，避免刷屏。

## 常用客户端命令

客户端命令不需要服务器 OP 权限。完整命令名为 `/netmusicbiliclient`，可使用短别名 `/ncpbc`。

| 命令 | 用途 |
| --- | --- |
| `/ncpbc status` | 查看媒体解析、编码和播放诊断信息 |
| `/ncpbc hologlass config` | 打开已佩戴全息眼镜的配置界面 |
| `/ncpbc pad cache status` | 查看 Pad 地图缓存状态 |
| `/ncpbc pad cache save` | 立即尝试保存 Pad 地图缓存 |
| `/ncpbc pad cache refresh` | 清空并重新生成 Pad 地图缓存 |
| `/ncpbc dolby joc on` | 开启 Dolby JOC 对象音频 |
| `/ncpbc dolby joc off` | 关闭 Dolby JOC 对象音频 |
| `/ncpbc dolby joc toggle` | 切换 Dolby JOC 对象音频 |
| `/ncpbc dolby joc status` | 查看 Dolby JOC 状态 |
| `/ncpbc dolby objects <0-64>` | 设置对象声源数量上限 |

## 常见问题

### 无法解析或刻录 B站视频

1. 确认输入中包含有效的 BV 号或 av 号。
2. 多 P 视频的分 P 序号必须是大于等于 1 的整数。
3. 在 NetMusic 电脑中重新扫码登录。
4. 多人服可能启用了链接白名单，请联系管理员批准视频。
5. 执行 `/ncpbc status` 查看诊断信息。
6. 查看游戏目录下的 `logs/latest.log`。

### 有声音但没有视频，或者播放卡顿

1. 把画质降到 720P、480P 或 360P。
2. 暂时关闭 Iris/光影包，判断是否存在渲染兼容问题。
3. 确保 MP4 或 Pad 位于快捷栏或副手，并处于活跃状态。
4. 完全退出游戏后，备份并删除 `config/net_music_can_play_bili/natives/`，重新启动游戏让模组释放原生解码库。

模组已经打包常见平台所需的原生解码库，普通游玩不需要另行安装系统 FFmpeg。

### Pad 地图不更新或显示异常

1. 执行 `/ncpbc pad cache status`。
2. 执行 `/ncpbc pad cache refresh`。
3. 仍有问题时，退出游戏，备份并删除游戏目录下的 `netmusic-pad-map-cache` 后重试。

### 耳机或全息眼镜没有输出

1. 确认装备位于头盔栏或 Curios 头部槽。
2. 使用媒体管理工具重新绑定。
3. 确认玩家与媒体设备处于同一维度且没有超出约 64 格。
4. 全息眼镜最多绑定 4 个媒体源，绑定已满时先解绑一个。
5. 检查唱片机是否因已连接音响或视频投影仪而触发互斥规则。

### 投影仪或音响设置没有生效

- 靠近设备再保存，建议保持在 8 格以内；
- 确认拥有该位置的建造权限；
- 检查客户端与服务端的模组版本是否一致；
- 出生点保护或领地插件可能阻止配置修改。

## 配置与隐私

客户端 Bilibili、Dolby 和登录配置通常保存在：

`config/net_music_can_play_bili.json`

该文件可能包含 Bilibili 登录 Cookie。**不要把它上传到公开仓库、网盘、聊天群或发送给其他人。**

服务端常用配置通常在首次启动后生成于：

`config/net_music_can_play_bili-common.toml`

其中可设置调试日志、链接白名单以及白名单拒绝提示中的管理员联系方式。

## 服务器管理员附录

完整服务端命令名为 `/netmusicbiliserver`，短别名为 `/ncpbs`。

| 命令 | 默认权限 | 用途 |
| --- | --- | --- |
| `/ncpbs sources` | OP2 | 查看活跃媒体源 |
| `/ncpbs sources limit <1-100>` | OP2 | 限制显示的媒体源数量 |
| `/ncpbs pad refresh` | OP2 | 刷新 Pad 服务端临时状态 |
| `/ncpbs whitelist add <BV/av/链接>` | OP4 | 添加链接白名单 |
| `/ncpbs whitelist list` | OP4 | 查看链接白名单 |
| `/ncpbs whitelist remove <BV/av/链接>` | OP4 | 移除链接白名单 |
| `/ncpbs whitelist export` | OP4 | 由玩家导出白名单 CSV |
| `/ncpbs whitelist review` | OP4 | 打开白名单审核界面 |

可供权限模组分配的节点：

- `net_music_can_play_bili.audit.sources`
- `net_music_can_play_bili.pad.refresh`
- `net_music_can_play_bili.whitelist.manage`

公网服务器建议启用链接白名单，并通过举报与活跃媒体源审计管理玩家播放的内容。

## 游玩限制速览

- 当前版本为 beta，建议在更新前备份世界和配置。
- MP4 最多装入 18 张唱片。
- Pad 最多保存 64 个媒体和 128 个点位。
- Pad 点位触发半径为 1～128 格。
